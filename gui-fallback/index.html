⚠ Integrity FAILED

This node's database has been marked as degraded. Sync is paused — no outgoing writes will be sent to peers, and this node will not accept incoming sync actions until recovery.

Common cause: A Force Restore was applied (disaster recovery). The restored database is flagged as degraded until a healthy peer pushes its full backup to this node.

How to recover

  1. Wait — on every boot the app requests a full backup from the first reachable peer. If peers are online, recovery is automatic within ~10–30 s of (re)starting the service.
  2. Restart the service — if the node has been running for a while without recovering, restarting triggers boot-catchup again: systemctl restart blueprints-app
  3. Manual restore — navigate to Settings › Backups and restore from a known-good backup, then restart the service.
  4. No peers available? — if this is the only node, you can reset integrity manually from the SQLite shell: UPDATE sync_meta SET value='true' WHERE key='integrity_ok'; Only do this if you are certain the data is uncorrupted.

⚠ Restore backup

Restoring:

This will replace the live DB on this node only. Other nodes are NOT updated. Their gen is higher, so they will push their current state back to this node on the next sync, overwriting this restore. Use Force Restore to propagate to peers.
⚠ FORCE RESTORE — this node's gen will be set above all peer gens. On the next sync drain, this node will push the restored state to ALL peers, overwriting their current data. Only use this for disaster recovery or corruption fixes.
