📚 About & API Reference

▶ How this extension works

This extension is a client for a Blueprints node — a self-hosted FastAPI service that maintains a personal bookmark and browsing history database. The extension never contacts any third-party service; all traffic goes exclusively to the node URL you configure above.


Three active components:

  • Popup — save the current tab as a bookmark or search your library via the toolbar icon.
  • Visit recorder (opt-in) — automatically logs every page you visit to the node so browsing history can be searched alongside bookmarks.
  • Context menu — right-click any page or link to save it directly.
▶ Authentication — TOTP scheme

Every API request (except /api/v1/bookmarks/health) must carry an X-API-Token header. The token is derived fresh for every request — the raw secret is never transmitted.


Token derivation (implemented in shared/api.js):

window_number = floor(unix_time_ms / 5000)   // changes every 5 seconds
token = HMAC-SHA256(secret_hex, str(window_number))
                                               // token is 64 hex chars (256-bit)

The server accepts the current window, the previous, and the next (±5 s skew tolerance). If the clocks between browser and server differ by more than ~15 s, authentication will fail intermittently.


Where to find your secret: BLUEPRINTS_API_SECRET in the .env file on your node. It is a 64-character hex string (256-bit random). Do not share it — anyone with it can read and write your full bookmark database.


IP allowlist: the node also checks that requests arrive from a permitted CIDR range (BLUEPRINTS_ALLOWED_NETWORKS in .env). If Test Connection returns HTTP 403, your browser's IP is likely not in the allowlist.

▶ TLS / SSL — certificate requirements

The node serves HTTPS using a TLS certificate. For the browser to trust it without an error, the certificate's CA must be installed in the browser's certificate store (or the OS trust store that the browser inherits).


What this extension assumes:

  • The API URL you enter uses https://.
  • The node's certificate is signed by a CA your browser already trusts — either a public CA (e.g. Let's Encrypt) or a private/self-signed CA that you have manually imported.
  • If you are using a private CA, you must install that CA's root certificate into your OS or browser trust store before the extension can connect. The extension cannot bypass certificate errors — that is intentional.

Diagnosing certificate problems:

  • A TLS error appears in the connection log as a network error (not an HTTP status code), typically before any HTTP headers are seen.
  • The log line will say something like: Failed to fetch or net::ERR_CERT_AUTHORITY_INVALID.
  • To confirm: paste your API URL directly into a browser tab — if you get a certificate warning there, you need to install the CA first.
  • No CORS headers will be present in the log when a TLS error occurs (CORS requires a completed HTTP response).

Port note: the standard HTTPS port 443 is used for browser and extension traffic. Port 8443 is used exclusively for node-to-node mTLS sync and is not reachable by the extension.

▶ API endpoints used by this extension

All endpoints are under {API_URL}/api/v1/bookmarks/. All require X-API-Token except where noted. All accept and return JSON.


GET /api/v1/bookmarks/health

Auth: exempt — no token required. CORS: open (*).

Used by: Test Connection button.

Returns overall node health. Response fields:

{
  "status":               "ok" | "degraded",
  "sqlite":               "ok",
  "seekdb":               "ok" | "error",      // full-text + vector index
  "embedding":            "ok" | "error",      // AI embedding service
  "embedding_error":      "",                  // empty if ok
  "bookmark_count":       4784,
  "visit_count":          0,
  "seekdb_indexed":       4784,                // should equal bookmark_count
  "seekdb_visits_indexed":0,
  "last_seekdb_sync":     "2026-03-20 10:17:42"
}

POST /api/v1/bookmarks/

Auth: required. Content-Type: application/json.

Used by: Save button in popup, context menu.

Request body:

{
  "url":         "https://example.com",  // required
  "title":       "Page title",           // optional, default ""
  "description": "",                     // optional
  "tags":        ["tag1", "tag2"],       // optional, default []
  "folder":      "",                     // optional
  "notes":       "",                     // optional
  "favicon_url": "",                     // optional
  "source":      "extension"             // optional, default "manual"
}

Returns: 201 Created with the full bookmark object (includes server-assigned bookmark_id, created_at, updated_at). Duplicate URLs are accepted (no dedup at this endpoint). The bookmark is replicated to all fleet peers automatically.


GET /api/v1/bookmarks/search?q=<query>&limit=15

Auth: required.

Used by: Search panel in popup.

Hybrid search: BM25 full-text + HNSW vector search, merged with Reciprocal Rank Fusion, then reranked by a language model. Both bookmarks and visit history contribute to results.

Query parameters: q (required), limit (default 20, max 100), include_visits (default true).

Response:

{
  "query": "your search",
  "count": 5,
  "results": [{
    "item_type":   "bookmark" | "visit",
    "id":          "seekdb internal id",
    "bookmark_id": "uuid or empty",
    "title":       "Page title",
    "url":         "https://...",
    "tags":        ["tag1"],
    "description": "",
    "notes":       "",
    "domain":      "example.com",
    "score_sources": ["bookmark_keyword", "bookmark_vector"],
    "rrf_score":   0.032
  }]
}

POST /api/v1/bookmarks/visits/

Auth: required. Content-Type: application/json.

Used by: Visit recorder (background service worker, only when enabled).

Request body:

{
  "url":            "https://example.com",  // required
  "title":          "Page title",           // optional
  "source":         "visit-recorder",       // optional
  "dwell_seconds":  null,                   // optional, int
  "visited_at":     null                    // optional ISO timestamp; server fills now() if omitted
}

Returns: 201 Created with the visit record. If the URL matches an existing bookmark, the visit is linked to it via bookmark_id.


GET /api/v1/bookmarks/extension-download

Auth: required (TOTP via GUI's apiFetch, not direct navigation).

Used by: Download Extension button in the Blueprints GUI and the Download update button on this page.

Returns: the extension as a application/zip file stream (blueprints-bookmarks-extension.zip).


GET /api/v1/bookmarks/extension-version

Auth: exempt — no token required. CORS: open (*).

Used by: service worker version check, version status bar on this page.

Returns the version of the extension build currently on the node. Response:

{
  "version":       "1.1.0",
  "download_path": "/api/v1/bookmarks/extension-download"
}
▶ Dependencies on the node

For full functionality the node must have the following services healthy:


Dependency Required for If missing
blueprints-app Everything All requests fail (connection refused / timeout)
SQLite DB Save, list, visit recording HTTP 500 on write endpoints
SeekDB (seekdb service) Full-text + vector search Search returns 0 results; save still works
AI embedding service Vector (semantic) search Search falls back to keyword-only; health shows "embedding":"error"

Checking status: the connection log shows the state of each dependency after a successful Test Connection. You can also check the node's journalctl -u blueprints-app and systemctl status seekdb for service health.
